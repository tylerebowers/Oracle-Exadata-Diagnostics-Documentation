main - Diagnostic Assistant: Exadata Main Menu 
   
  The Oracle Diagnostic Assistant Exadata Main Menu enables you to collect
  diagnostics based on a specific issue. Exadata system components have been
  grouped by functional categories below and includes access to Exachk.
    
  Select one of the following problem categories:
    (1) Database
    (2) Exadata - Cell
    (3) Exadata - Hardware
    (4) Network
    (5) Oracle Errors
    (6) Processes
    (7) RAC
    
    ----------Other Diagnostic Tools ----------
    (11) Exachk
    (12) OSWatcher

    (H)elp    (E)xit

Showing Menu Help: 
  Oracle Diagnostic Assistant Exadata Main Menu Help 
     
  The Exadata Main Menu enables you to diagnose your Exadata environment.
   
  Explanation of available options:
     
    (1) Database
        o Select this option if you suspect a database issue
 
    (2) Exadata - Cell
        o Select this option if you suspect a cell failure, performance
          issue, or patching problem

    (3) Exadata - Hardware
        o Select this option if you suspect underperforming flash drives
          or failed drives on Exadata Cell or Compute Node

    (4) Network
        o Select this option if you suspect issues with the listener, Network
          cabling, IB Switch or any other Network related problem 

    (5) Oracle Errors

          a process you suspect is spinning/hanging using procwatcher

          the Diagnostic Assistant

    (7) RAC
        o Select this option if you suspect issues with the RAC instance or
          node evictions or you encounter clusterware startup issues

    delivered with the Diagnostic Assistant, but are commonly used to
    troubleshoot issues related to the menu target.

    (12) OSWatcher
         o Access to OSWatcher archive files

         o If you select this option, use (M)ain to return to this menu

DAexaCellILOMGen - Generate ILOM Snapshot 
  You must run the ILOM Snapshot outside the Diagnostic Assistant
  at this time. 
  Please perform the following steps in order given.
  Detailed instructions for each step are available in (H)elp.

    1. ssh to [node name]-ilom and enter the password
    2. Confirm header details
    3. Run command: set /SP/diag/snapshot dataset=normal
    4. Confirm response
    5. Run set command to set output directory
    6. Run command: cd /SP/diag/snapshot
    7. Type command: show
    8. Verify the status under properties is 'Running'
    9. When Snapshot Complete, end ILOM session

    (M)ain    (B)ack    (H)elp    (E)xit

[+] Toggle Help

Showing Menu Help: 
  ILOM Issues: Generate ILOM Snapshot Help

  Explanation of available options:
    1 ssh to the [node name]-ilom and enter the password at the prompt
       example: ssh cell01-ilom
                password
    2 You should see a header similar to the following output:
        Sun(TM) Integrated Lights Out Manager
        Version 3.0.9.19.a r55943
        Copyright 2009 Sun Microsystems, Inc. All rights reserved.
        Use is subject to license terms.

    3 After the prompt,type this command: 
        set /SP/diag/snapshot dataset=normal

    4 You will get a response of: 
        Set 'dataset' to 'normal'

    5 After the prompt, type this command:
        set /SP/diag/snapshot dump_uri=sftp://root:[password]@
          [IP address of host]/[directory to upload snapshot to]
        ex: set /SP/diag/snapshot dump_uri=sftp://root:
            passwd@192.168.1.7/tmp
 
    6 After the prompt, type this command: 
        cd /SP/diag/snapshot

    7 Type command: show

    8 Verify that the status shown under properties is 'Running'
        example: show
                 /SP/diag/snapshot
                 Targets:
                 Properties:
                    dataset = normal
                    dump_uri = (Cannot show property)
                    encrypt_output = false
                    result = Running
 
    9 Continue to enter the "show" command until these lines 
      appear under Properties, indicating the process is complete:
        result = Collecting data into [sftp location from step 5]
        Snapshot Complete.
        Done.

   10 To end the ILOM session, type command: exit

    See My Oracle Support Knowledge Article 1062544.1: Diagnostic 
    information for ILOM, ILO , LO100 issues (https://support.oracle.com/
    CSP/main/article?cmd=show&type=NOT&id=1062544.1) for additional 
    information.
