  troubleshooting and gathering of relevant diagnostics.

    (1) Configure
    (2) Install
    (3) Performance

[+] Toggle Help

Showing Menu Help: 
  Oracle Diagnostic Assistant Host Target Main Menu
 
  The Diagnostic Assistant Host target menu enables targeted  
  troubleshooting and gathering of relevant diagnostics.
  
  Explanation of available options:
  
    (1) Configure
        o Executes Oracle Configuration Manager OCM to collect Host data
        o Executes RDA to collect operating system data
        o Executes RDA tool TLsecure to identify potential security risks
         
    (2) Install
        o Executes Health Check / Validation Engine HCVE
        o Executes Remote Diagnostic Agent (RDA) to collect operating system  
          and Oracle installation data
              
    (3) Performance
        o Executes RDA profiles and modules to collect performance data
        o Executes RDA tools TLsos, TLjroc, and TLepm to collect 
          siga/sosreport/sysreport, JRockit process, and Enterprise 
          Performance Management Validation information

Showing Menu Help: 
  Oracle Diagnostic Assistant Host Target: Install Menu Help
    
  Explanation of available options:
    
    (1) Health Check Assessments 
        o This option shows a menu of available pre-installation healthcheck 
          assessments using the Healthcheck Validation Engine (HCVE)
        o Healtchcheck Assessments are performed by the Healthcheck Validation 
          Engine (HCVE) and the HCVE rule engine is a tool that runs 
          inside Remote Diagnostic Agent Framework
        o The purpose of HCVE is to provide a generic engine to execute 
          assessment checks for customers and on the Oracle server side 
          environment using very abstract rules provided by the product experts

        o RDA will execute the following module:
          o OS (collects operating system information)
          o NET (collects network information)

    (M)ain    (B)ack    (H)elp   (E)xit

[+] Toggle Help

Showing Menu Help: 
  Oracle Diagnostic Assistant Host Target: Configure Help
   
  Explanation of available options:
  
    (1) OCM Host data - TBD
        o This option will collect OCM Host data 
  
    (2) Operating System Information
        o This option execute the OS module within RDA

    (3) Identify potential security risks
        o This option will run RDA:TLsecure for all security risks including:
          - Users with a default password (11g)
          - Users with a well known password
          - Users not visible in dba_users
          - Operating system authenticated user names with a password
          - Users with SYSTEM as default tablespace
          - Users and privileges from gv$pwfile_users
          - Privileges not granted by their owner
          - Listeners without a password
          - Listeners with local operating system authentication 
            (10g and later)
          - Listeners modifiable at runtime
          - AUTHENTICATION_SERVICES value in sqlnet.ora files
          - Oracle executables owned by different users of a same group 
            (UNIX only)

Showing Menu Help: 
  Oracle Diagnostic Assistant Host Target: Networking Help
   
  Explanation of available options:
    
    (1) Network Information
        o This option executes the NET module within RDA
  
    (2) Network Advisor Tool - TBD
        o Net Advisor is a tool that provides advice on resolving network layer 
          errors encountered using Oracle Net Services. 
          Inputs:
            - the absolute path of a valid Oracle Networking trace file
        o The tool will analyze the trace file. It notes which naming methods 
          were used in attempting to resolve a given service name and 
          provides an overview of the network configuration. 
          Net Advisor identifies conflicts or issues in the network setup of 
          the client, and suggests what to check or which actions to take 
          for the issues found. Operating System data will be collected.
