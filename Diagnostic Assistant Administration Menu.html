    The Administration Menu enables you to perform limited

    Explanation of available options:
    (1) Manage Properties
        This selection provides options for viewing and managing property values
        There are currently 2 property files used by the Diagnostic Assistant:
            o da.properties which defines common properties 
            o user.properties has limited functionality - it is 
              possible that this file is empty

              time within the tool (Ie. MOS Username, CSI)

        This option provides access to view summary and detailed information
        of data collections performed

        summary information.  Also, a list of a diagnostic collections 
        executed during the current session is available for viewing.
			   
    (3) Diagnostic Assistant Version
        This option displays the Diagnostic Assistant version number, 
        build number and build date.
		Also included are the versions of ADR, RDA ad OCM (if they exist)
		
        This information is helpful when logging a service request for
        the Diagnostic Assistant or in debugging issues with
        Development.
		
        Make sure you have the latest version of the tool, by
        comparing your version with latest version documented in 
        MOS knowledge article 201804.1.

AdminProp - Manage Properties 
  
    The following options allow for viewing the properties    
    files and managing the stored properties.
      
    Select one of the following options:
      (1) View da.properties
      (2) View user.properties 
      (3) Reset a Required Setup Property
      (4) Change MOS Password 
      (5) Allow/Disallow saving passwords
      (6) Reset configured properties 
      (7) Restore ALL Stored Properties back to original default

      (M)ain    (B)ack    (H)elp   (E)xit

[+] Toggle Help

Showing Menu Help: 
    Administration: Manage Properties Menu Help
    
    Explanation of available options:
	
          (1) View da.properties
              o displays da.properties file contents
              o results will display on the screen page by page
			   
          (2) View user.properties
              o displays user.properties file contents
              o currently this file is used for one property only
                - Procwatcher directory path
              o additional values will be addded as functionality expands
             
          (3) Reset a Required Setup Property
              o Permits reset of values stored in da_properties.ini
              o allows change to the values in the event they were
                incorrectly entered when initially prompted

          (4) MOS Password
              o Once prompted, this value is stored in an Oracle wallet file
                and will not be prompted for again
              o this option allows MOS password to be updated in the event 
                it was incorrectly entered when initially prompted
              o MOS password is used when creating a draft SR and uploading 
                diagnostic files
              
          (5) Allow/Disallow saving passwords
              o By selecting this option, the 'use_wallet' property can
                be changed to TRUE (which allows passwords to be saved in a 
                secure Oracle wallet) or FALSE (which does not allow
                passwords to be saved, and so the user is prompted each time).
		  
              o This property is set to FALSE by default
              
          (6) Reset configured properties
              o This loops through each user modified da.properties
                one by one
              o If no change is needed, hitting return will retain current
                setting
                       
          (7) reset all saved parameters
              o This deletes any DA.* entries, and updates the following:
                  * use_wallet=false
                  * wallet.key=
                  * browser_viewer_spawned=false 
				  
              Note: Oracle wallet is used to secure passwords

Showing Menu Help: 
    Exadata Detailed Diagnostics: Core Collectors -  RDA, OCM, ADR and More Help   
    
    Explanation of available options:
     
      (1) ADR - Automatic Diagnostic Repository
            o Selecting ADR will enable you to view ADR files
            o No login is required for ADRCI because the information stored 
              in ADR is not intended to be secure
            o ADR is a special repository automatically maintained by 
              Oracle 11g to hold diagnostic information about critical 
              error events
         
      (2) OCM - Oracle Configuration Manager
            o Selecting OCM will enable you to view and collect the latest
              configuration data 
            o OCM is a proactive utility linked to My Oracle Support that 
              collects configuration data specific to your environment
            o OCM includes healthchecks, patch advice and recommendations
            o When configuration issues are detected, clear and actionable 
              recommendations are presented, enabling prompt and effective 
              remediation of the issue(s) 
          
      (3) RDA - Remote Diagnostic Agent
            o Selecting RDA will enable you to view the RDA
              collection files, run RDA module(s), and run RDA profile(s)
            o Remote Diagnostic Agent (RDA) is a command-line diagnostic 
              tool that includes a suite of data collection and diagnostic
              scripts that provides a comprehensive picture of the 
              customer's environment
		  
      (4) Procwatcher
            o Procwatcher is an external tool to examine and monitor Oracle
              database and/or clusterware processes at an interval
            o If you experience problems with Procwatcher, please contact  
              the owner of My Oracle Support Knowledge Article 459694.1: Script 
              to Monitor and Examine Oracle DB and Clusterware Processes
              https://support.oracle.com/CSP/main/article?cmd=show&type=
              NOT&id=459694.1) for assistance
            o Select this option to perform one of the following functions:
               (1) View Procwatcher status
               (2) Start Procwatcher
               (3) Stop Procwatcher
               (4) Package Procwatcher files
			   
      (5) OSWatcher
            o Perform grep commands on various OS Watcher log files
